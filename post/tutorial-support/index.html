制作教程用到的软硬件清单

· 2020-09-20 · # 日常

(10 min 40 s reading) 我如何在现有基础上尽可能减少开销并进行教程制作。



因为 WFH (work from home) 开始做视频教程，横向对比网上相关（少得可怜的）中文资源，自认为做出来的东西还算及格分以上。虽然说内容是决定视频质量的第一要素，但围绕着内容可以做出努力，使得体验更好。作为（伪）学生党，我对这方面的投入算是比较节省。在这里总结一份清单来介绍下这几个月做 XAFS 教程用到的软件/硬件。

硬件

1. 画面：Nividia GTX 1070 + Dual 4K Monitors

几年前购买了台式机后，它就成为了我的主要生产力工具。 花了快一万人民币买的 ThinkPad X1 Carbon 在家沦为看B站的工具 教程均是在台式机上操作完成的。笔记本尤其是超极本，在我看来不太适合制作视频教程，一方面是配置确实受限，另一方面屏幕太小，可展示的空间不尽如人意。读博期间能有个台式机处理数据和写论文，我认为可以排 Todo List Top 3。

提到N卡是因为它自带 Nvidia Geforce Experience 可以方便录屏，只需要一个 Alt + F9 就开始了，内存占用不多。类似的竞品有 Xbox Game Bar （可以用 Win + G 启动），但是试用之后我觉得没有N卡自己带的方便，而且占用的内存较多。我做教程的时候经常是很多个浏览器和软件全开，而且两块屏幕 (27 + 28) 均是 4K，所以个人更推荐N卡自带的软件来录屏。最近 Nvidia 新发布的 GTX 3000 series 可以做到降噪让我十分心动（有人在旁边用吹风机都能降噪到一点声音没有的那种），个人认为不缺预算上 GTX 3080 是很香的，但只是做个小教程，用用普通N卡也是很不错的，蹭一个软件不亏。1070 这张卡还是我男朋友换上 2080 剩给我的，所以开销为 0。

4K 显示器这个无论是做教程还是出于工作需求都是强烈推荐的，不差钱至少上 27 寸以上 4K 可调高度的那种，最好是 Dell 的，能有多好就有多好。预算紧张的有 24 寸的 1080 IPS 显示器也很香，我黑五买过全新 1080p IPS 27寸 Dell 显示器不到 $80，收个二手也是极好的。屏幕大了之后可以更容易地把内容展示出来，小屏幕会看的比较难受。而且还要涉及到后续编辑视频，大屏幕会让眼睛没那么快觉得紧张疲劳。

对于一个软件视频教程而言，画面是传递信息最直观的方式。如果无法保证视频的清晰度，视频内容质量很受伤害。



2. 声音：Blue Microphones Snowball iCE + Pop Filter

如果需要把自己声音放进去，请一定要有专门的麦克风。

这是我看过B站化学相关的知识区视频之后得到的重要结论，但很多 UP 主没有留意过这个细节。很多视频画面方面做得非常不错，还有动画转场字幕等等，但是声音这一块听起来真的不愉快。我通常做视频教程的时候想象的是：在教室里老师讲台上的样子。除开板书，老师们会通过声音来补充很多知识点。我认为视频教程亦是如此：利用声音给教程增添信息，但不能让人分心。

最重要的是发音清晰，而麦克风能够准确捕捉到我的声音。在一开始的视频我使用的是 webcam 上自带的麦克风，够用但是差强人意，所以入了 Blue 家的麦克风。买的是最便宜的那种 ($49)，预算往上走还有它家最有名的 Yeti 系列麦克风，支持更多的收音模式和更高的音质。我不建议使用笔记本自带的收音，因为人坐的位置离笔记本的拾音口太远，而且很容易把杂音给收进去。我也不认为一定非得买 Blue 家的麦克风，收音干净的就可以，离自己嘴巴比较近并且有心形收音功能更好，因为这样麦克风就可以尽可能地保留人声而降低对其他方向的杂音。

录音时，建议使用单声道录制。Yeti 系列麦克风默认是双声道收音，虽然功能更强大了，但在视频教程里有点弄巧成拙。因为在录制教程的过程，常常脑袋会晃来晃去看电脑和台本（教程都是有台本准备再开始录制的），如果使用双声道录制而观众用耳机收听的时候，就会明显感觉得到我的声音是在晃动，这个听感对视频内容没有任何帮助，甚至是一个让人分心的行为。而利用单声道录制时，由于是心形收音曲线，最多只会感觉到有一点音量起伏，听感影响不大。

使用 pop filter 或者叫“兔耳”的配件是为了减少爆破声，呼吸声还有吧唧嘴声音的输入。由于麦克风离嘴巴比较近，如果念到爆破音（ papi、呸等等）音轨线就会变形得到一个爆音，拿耳机听会特别难受。加入一个 pop filter 或者是兔耳就可以减少爆音出现还有呼吸音等杂音，不需要花大钱就能提升听感一大利器。



3. 文字：Cherry G80-3000 Keyboard + iPad Pro 2017 + Logitech Key-to-go

文字输入最关键的就是键盘了，机械键盘里面用来用去还是 Cherry 经典款最舒服，原价比较贵的(~$100)。我运气好从别人那里花 $30 收了一个几乎全新的 G80-3000。长时间输入还是要依赖机械键盘。无论笔记本键盘还是 Mac 键盘，都是受刑体验。机械键盘还算长寿，遇到别人出二手的时候收一个，香。但如果你和室友同住宿舍，要留意机械键盘发出来的声音可能会不利于室友之间的友谊。

不是所有的文字工作都是在台式机前进行的，有时候有灵感了就会坐下来抓出 iPad Pro 开始写 outline。移动端中，文字输入是 iPad + 外接键盘，选择 Logitech Key-to-go 完全出于便携的原因，要说跟机械键盘比输入感那是完败。如果对便携要求不那么严格，Logitech K380 可以更好地平衡输入体验和便携性，价格也要低很多。




软件

1. 画面：Hitfilm + GoodNotes + Notability

除开N卡自带的 Nvidia Geforce Experience ，我只使用 Hitfilm 来编辑视频。原因嘛，不要钱。只需要花几个小时的时间看一看教程，然后就能把视频基本剪出来的，对新手友好。 要说缺点就是对中文文本支持不太行，有时候打字打到一半就崩溃了。还有个主要拿来给视频打码，总的来说使用频率不算高，够懒。（最近想尝试 Adobe 全家桶，但价格实在是太贵了哈哈。）

我个人不喜欢使用 PPT 来做教程，因为极为无趣而且传递不了跟软件相关使用的小细节。 在选择讲一些理论相关知识点的时候，我倾向于做板书，是为了让听众有足够的时间跟上我的节奏，而我也可以在其中灵活插入补充点。而要做成视频，iPad Pro + Apple Pencil + 笔记软件无疑是完美组合。GoodNotes 和 Notability 是知名老牌 IOS 端笔记软件了，平常就很喜欢拿来打草稿或者是远程开会涂鸦。要说缺点就是录屏不是 16:9，会有一部分屏幕留白，不过影响不大。



2. 声音：Audacity

因为教程涉及到了视频中的音频还有播客，有一个音频软件是必需品。选择 Audacity 的原因还是：免费。 功能足够强大，可以剪辑可以修音，学习成本不高，Mac/Windows 均可使用。

我常用功能有 Normalization （把声音给拉到比较合适音量）、Noise Reduction （背景噪音）、Amplify （解决爆音）、Noise Gate （解决呼吸声），有兴趣的可以在油管上搜索相关教程。



3. 文字：Gridea + Typora + Bear

文字教程相关都是使用 Markdown 产出。作为一个非码农选手，Markdown 的学习成本比 LaTex 要低很多，还能有非常不错的排版。现在有不少文字平台已经支持 Markdown 语法，比如简书和印象笔记。如果你想了解有关 Markdown 的使用，在少数派上可以找到不错的介绍文章，比如 「Markdown 完全入门」。

基于 Markdown 的语法使用，我在 Windows 平台选择 Typora 输入，其开源免费界面美观的优点可以几乎无脑选择。可以阅读少数派 2015 年的文章 「让 Markdown 写作更简单，免费极简编辑器：Typora」 来了解它的优点。然而 Typora 没有移动端，于是我选择了 Bear （中文名熊掌记）作为 IOS 端 Markdown 输入。在另外一篇文章里介绍过这个软件，虽然是订阅制收费，但架不住软件的美貌。除开 Bear 以外还有竞品 Ulysses 很多人使用。

Gridea 也是我在少数派上找到的免费静态博客客户端，依托于 Github Pages 账号搭建个人网页非常方便，省去了学习前端的时间，几乎是可以傻瓜操作。出于能省则省，且不要搞得花里胡哨的目的，我没有使用 WordPress。然而使用这样比较小众客户端搭建网页，也会有不少缺点，比如我目前网页搜索功能几乎是不能用，得等男朋友忙完这一阵之后帮我写一写。我真的对代码一窍不通。




总的来说，因为做教程的原因产生出来的开销大头还是在麦克风上，其他的硬软件都是因为非教程目的早已购入或者是能用免费的就不用收钱的。我一开始看别人制作的教程，总感觉别人用的工具都是高大上，我完全不能承受的那种。后来也就发现似乎不是这么一回事，这中间离不开男朋友的技术支持 还有B站三连支持。对于比较专业的软件视频教程，内容的打磨和主题的选择是王道，很多UP主视频讲解专业软件非常精彩，即使是音频或其他方面有点小瑕疵也并不影响播放量。如果有一天你想要开始制作教程，我推荐先看看别人的作品，去仔细推敲别人的优缺点，寻找到自己独一无二的竞争力。而今天推荐的软硬件其实只是一个“锦上添花”的作用。
